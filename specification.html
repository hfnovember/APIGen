PaNick Apps API Generator Specification

What it is

The PaNick Apps MySQL-PHP API Generator is a powerful tool designed to automate the construction of entity classes and APIs from MySQL databases. The generator can be used to create API endpoints in PHP from database tables and their fields. There is a standard set of default endpoints for each table/entity that supports basic operations such as creating and removing elements and more. In addition, custom endpoints can be created which can then be implemented accordingly by the programmer. The generator supports automatic generation of Java classes which can be used to retrieve data from the API on a client application as well as automated security and authorization checks, parameter checks and informational responses on issues regarding the call. Thus, the PaNickApps API generator can cut development time substantially especially for projects involving a lot of data.

PHP class generation (server-side)

To assist the programmer in creating their own endpoints, PHP classes can also be generated from the database tables. These entity classes convert database fields into class attributes with the proper set/get methods and constructors. In addition to these, several static methods are also generated for each class. These methods can be used to automatically perform actions on the database such as inserting or deleting elements of their type without having to write SQL code.

Java class generation (client-side)

The generator can also be used to provide the user with automatically generated Java classes. These are similar to the generated entity classes for PHP without the static methods which cannot be used at a client level. These classes are generated in an identical way and are raw Java classes without any library dependencies. These Java classes can be used to develop any Java application such as Android applications, Desktop applications or other services.

Automated security

The PaNickApps API Generator supports automated security checks! Every API endpoint created by the generator automatically supports security checks for authentication of users. The generator uses pre-defined tables in your database to automatically set up and destroy sessions for client applications. It utilizes these sessions to allow each group of users to only access endpoints which they are allowed to access. This means that you do not have to worry about any authentication processes taking place in your API. Instead, you will be asked to select which users can access which endpoints of the API. Using these selections, the generator will automatically create the default access rights for each user on each of these endpoints.

Automated status & parameter checks

In addition to security, the generator provides each endpoint with automated informative messages to display to users when something was done or has gone wrong. For example, each endpoint can notify the caller application with appropriate messages to report the status of the call. Messages include any technical difficulties on the server or database, missing any parameters to the call or trying to access an endpoint while not authorized.

Database consistency requirements

The generator requires that a database contains several specified tables with pre-defined fields to automate authentication, authorization and security. There are three tables required in an acceptable database with the following fields:

  • Users Table:
    • UserID - Integer, Auto-Increment, Primary Key
    • Username - Varchar(255), Not null
    • Password - Varchar(255), Not null
    • UserLevelID - Varchar(255), Not null

  • UserLevels Table:
    • UserLevelID - Integer, Auto-Increment, Primary Key
    • UserLevelName - Varchar(255), Not null

  • Sessions Table:
    • SessionID - Varchar(255), Primary Key
    • UserID - Integer, Not null
    • InitiatedOn - Timestamp, Not null
    • FinalizedOn - Timestamp

Option 1: Creating a new database

Creating a new database using the generator is by far the easiest option. The generator can automatically create a new database for you with the above tables.

Option 2: Modifying an existing database

In the (unlucky) case that you already do have a database you do not want to abandon, it is suggested that you make a backup first before moving on. To make your database consistent with the generator you must make sure that any tables with the same names as above are either re-created with the above fields or the above fields are added to these tables. If any already existing fields have the names of the pre-defined fields, you must rename them before continuing. Simply put, these tables, with these specific fields, with these specific types and properties MUST exist in the database. Adding other fields in these tables will not affect the generator.

Supported data types

The API Generator safely supports the following MySQL data types:

  • Integer
  • Float
  • Double
  • Char
  • Varchar
  • Text
  • Longtext
  • Date
  • Time
  • Boolean

It is strongly suggested to not use any data type names as column/field names. Using other data types may or may NOT work depending on the type. Feel free to try any other MySQL type but use them at your own risk!

Rules

  • All tables should have exactly 1 primary key. Take special caution in joiner tables.
  • Table names should start with an uppercase letter because they are later converted into classes.
  • Do not mess with the pre-defined tables outside of the generator.
  • Endpoints support POST HTTP requests only.

Conventions

  • Use the suggested data types.
  • In case of time or date, it is suggested to use integers or long integers to store timestamps.
