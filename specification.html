PaNick Apps API Generator Specification

Database schema

After generation of the database or prior to generation of the login script the database should consist of at least the two following tables:

  • Users Table:
    • UserID - Integer, Auto-Increment, Primary Key
    • Username - Varchar(255), Not null
    • Password - Varchar(255), Not null
    • UserLevelID - Varchar(255), Not null

  • UserLevels Table:
    • UserLevelID - Integer, Auto-Increment, Primary Key
    • UserLevelName - Varchar(255), Not null

These tables should be automatically generated into the named database if you opt to create a new database.

Supported Data Types

Important Note: Do not use data type names as column names!

The API Generator safely supports the following data types:

  • Integer
  • Float
  • Double
  • Char
  • Varchar
  • Text
  • Longtext
  • Date
  • Time
  • Timestamp
  • Boolean

You may use other data types, but be aware that they may not work. It is also strongly suggested to use Timestamp types for date-time values.
